## To study:

1. storage algorithms and data structures (trees, hashes, tries)
2. practical storage peculiarities on each OS you plan to support. POSIX api on writing, flushing data, mmap-ing, O_DIRECT flags etc.
3. network and sockets: presumably you'd want to talk to your database via a socket so that can become a problem or performance bottleneck
4. concurrency (at thread/machine level): locks, threads, read/write barriers ...
5. distributed databases (if you plan to "scale in the cloud (tm)") ...
6. understanding CAP theorem at the very least.
